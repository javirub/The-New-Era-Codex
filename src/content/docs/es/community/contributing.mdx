# Contribuir a The New Era Codex

¡Bienvenido! Gracias por tu interés en contribuir a The New Era Codex. Esta guía te ayudará a entender nuestro enfoque único de múltiples audiencias y cómo crear contenido que sirva a las personas adecuadas de la manera correcta.

## Entendiendo nuestro enfoque multi-audiencia

The New Era Codex es único: organizamos el contenido **primero por audiencia**, no por tema. Esto significa que el mismo tema (como "usar IA para automatización") será completamente diferente dependiendo de para quién estés escribiendo.

## Antes de empezar

<Steps>

1. **Elige tu audiencia**

   Pregúntate: "¿Quién se beneficiará más de esta guía?" Tu respuesta determina:
   - En qué directorio colocar tu archivo
   - Qué tono y profundidad usar
   - Qué ejemplos incluir
   - Qué conocimiento previo asumir

2. **Elige la plantilla correcta**

   Proporcionamos plantillas para cada tipo de audiencia:
   - [Plantilla de Guía para Desarrolladores](/es/community/templates/developer-guide-template/)
   - [Plantilla de Guía de Automatización](/es/community/templates/automation-guide-template/)
   - [Plantilla de Guía para Todos](/es/community/templates/everyone-guide-template/)
   - [Plantilla de Guía para Profesionales](/es/community/templates/professional-guide-template/)

   Copia la plantilla apropiada y complétala con tu contenido.

3. **Revisa la guía de estilo**

   Consulta nuestra [Guía de Estilo](/es/community/style-guide/) para:
   - Convenciones de formato
   - Pautas de tono para cada audiencia
   - Cómo usar componentes de Starlight
   - Mejores prácticas de Markdown

4. **Configura tu entorno**

   ```bash
   # Haz fork y clona el repositorio
   git clone https://github.com/javirub/The-New-Era-Codex.git
   cd The-New-Era-Codex

   # Instala dependencias
   bun install

   # Inicia servidor de desarrollo
   bun dev
   ```

</Steps>

# Escribe mejores prompts de ChatGPT

## Por qué importa esto

¿Alguna vez le has preguntado algo a ChatGPT y obtenido una respuesta vaga e inútil? No estás solo. El secreto no es ChatGPT, es cómo preguntas.

En los próximos 5 minutos, aprenderás una fórmula simple que funciona siempre.

## El problema: prompts vagos obtienen respuestas vagas

### Prompt malo
```
Escríbeme un post de blog sobre productividad
```

**Lo que obtienes**: Un artículo genérico y aburrido que podría ser sobre cualquier cosa

### Prompt bueno
```
Escribe un post de blog de 500 palabras sobre consejos de productividad para trabajadores remotos con niños pequeños. Usa un tono amigable y conversacional. Incluye 3 estrategias específicas con ejemplos. La audiencia son padres trabajando desde casa que luchan con interrupciones.
```

**Lo que obtienes**: Un artículo enfocado y útil que realmente ayuda a la gente

## La fórmula de 5 minutos

Cada buen prompt necesita estas 4 partes:

1. **Qué** quieres (post de blog, email, resumen, etc.)
2. **Para quién** es (principiantes, expertos, padres, etc.)
3. **Cómo** debe sonar (formal, casual, divertido, etc.)
4. **Detalles** (longitud, ejemplos, formato, etc.)

## Pruébalo tú mismo

### Ejemplo 1: Escribir un email

**Malo**: "Escribe un email a mi jefe"

**Bueno**: "Escribe un email profesional a mi jefe solicitando el próximo viernes libre. Lo necesito para un evento familiar. Mantenlo breve y cortés, alrededor de 100 palabras."

**Copia esta plantilla**:
```
Escribe un email [tono] a [destinatario] sobre [tema].
[Cualquier requisito específico]. Mantenlo [longitud] y [estilo].
```

# Screening de candidatos potenciado por IA

## Impacto empresarial

**Tiempo Ahorrado**: ~2 horas por posición
**Reducción de Costos**: ~$150 por contratación en tiempo de RRHH
**Mejora de Calidad**: 40% más consistencia en evaluación de candidatos

## Para quién es esto

Esta guía está diseñada para profesionales de RRHH y reclutadores que:
- Revisan 20+ currículums por posición
- Luchan con criterios de evaluación inconsistentes
- Necesitan reducir el tiempo de contratación
- Quieren mantener cumplimiento con regulaciones de contratación

## El problema

El screening tradicional de currículums es:
- Consume tiempo (promedio 6 minutos por currículum)
- Inconsistente (diferentes revisores usan diferentes criterios)
- Sesgado (sesgos inconscientes afectan decisiones)
- No escalable (100+ aplicantes = días de trabajo)

## La solución

Usa IA para crear un screening inicial consistente y rápido que:
- Aplica los mismos criterios a cada candidato
- Marca los mejores matches para revisión humana
- Documenta razonamiento para cumplimiento
- Reduce tiempo de screening en 70%

## Cómo funciona

### Paso 1: Define tus criterios

Trabaja con hiring managers para listar calificaciones imprescindibles y deseables:

**Imprescindibles (Deal-breakers)**:
- 3+ años en el rol
- Certificaciones específicas
- Habilidades requeridas

**Deseables (Puntos bonus)**:
- Experiencia relevante en la industria
- Idiomas adicionales
- Experiencia en liderazgo

### Paso 2: Crea tu prompt de screening

```
Analiza este currículum para una posición de [Título del Trabajo].

Requisitos imprescindibles:
- [Requisito 1]
- [Requisito 2]
- [Requisito 3]

Calificaciones deseables:
- [Calificación 1]
- [Calificación 2]

Proporciona:
1. Puntuación de match (0-100)
2. Resumen breve de experiencia relevante
3. Preocupaciones potenciales
4. Recomendación (Sí Fuerte/Sí/Quizás/No)
```

### Paso 3: Revisa evaluaciones de IA

[Incluir ejemplo de output]

## Consideraciones importantes

### Privacidad y cumplimiento

- **GDPR/Privacidad de Datos**: Asegura consentimiento del candidato para procesamiento por IA
- **Anti-Discriminación**: La IA debe evaluar calificaciones, no demografía
- **Documentación**: Mantén registros de criterios de evaluación para auditorías
- **Revisión Humana**: La IA filtra, los humanos deciden; nunca automatices decisiones finales

### Mejores prácticas

1. **Valida Regularmente**: Audita recomendaciones de IA trimestralmente
2. **Entrena a Tu Equipo**: Asegura que RRHH entienda cómo funcionan las evaluaciones de IA
3. **Sé Transparente**: Dile a los candidatos que la IA es parte del proceso
4. **Itera**: Refina criterios basándote en resultados de contratación

## Qué buscamos

- **Preciso**: Prueba tu código, verifica tu información, cita fuentes
