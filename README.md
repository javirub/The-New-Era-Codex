## ğŸ“¦ Deployment

The New Era Codex is configured for easy deployment to **Cloudflare Pages**.

### Deploy via Cloudflare Dashboard (Recommended)

1. **Push to GitHub**: Ensure your code is pushed to a GitHub repository
2. **Connect to Cloudflare**:
   - Go to [Cloudflare Dashboard](https://dash.cloudflare.com/) â†’ **Workers & Pages**
   - Click **Create** â†’ **Pages** â†’ **Connect to Git**
   - Select your repository
3. **Configure Build Settings**:
   - **Framework preset**: Astro
   - **Build command**: `bun run build`
   - **Build output directory**: `dist`
   - **Node version**: 18 or higher
4. **Deploy**: Click **Save and Deploy**

Cloudflare will automatically build and deploy your site. Subsequent pushes to your main branch will trigger automatic deployments.

### Deploy via CLI

```bash
# Build and deploy to Cloudflare Pages
bun run deploy

# Or manually:
bun run build
bunx wrangler pages deploy ./dist
```

On first deployment, you'll be prompted to:
1. Authenticate with Cloudflare
2. Select your Cloudflare account
3. Name your project (default: `the-new-era-codex`)

### Custom Domain Configuration

The site is currently configured for `codex.javirub.pro`. To change this:

1. **Update the site URL** in `astro.config.mjs`:
   ```javascript
   export default defineConfig({
     site: 'https://your-domain.com',
     // ...
   })
   ```

2. **Configure in Cloudflare Dashboard**:
   - Go to your Pages project â†’ **Custom domains**
   - Add your domain
   - Update DNS records as instructed

3. **Rebuild and redeploy** to apply changes

### Automatic Sitemap

The site automatically generates a sitemap at `/sitemap-index.xml` using the configured `site` URL. This improves SEO and helps search engines discover your content.
